# teleport-rs — Error Handling

## Two-Tier Error Model

Errors are split into **transport errors** (network/serialization failures) and **application errors** (business logic failures from Rust). This separation exists because:

1. Transport errors can happen on any call — they are not specific to a procedure
2. Application errors are procedure-specific — `LoginError` is different from `GetUserError`
3. TypeScript needs to handle these differently (retry transport errors, pattern match app errors)

## Rust Side: `AppError<T>`

```rust
use serde::{Serialize, Deserialize};
use specta::Type;

/// The shared error type for all teleport procedures.
/// T is the procedure-specific error detail. Defaults to () for procedures
/// that don't need custom error types.
#[derive(Debug, Clone, Serialize, Deserialize, Type)]
pub enum AppError<T = ()> {
    /// Not authenticated / session expired
    Unauthorized,

    /// Authenticated but not allowed
    Forbidden,

    /// Resource not found
    NotFound,

    /// Bad request / validation error
    BadRequest(String),

    /// Internal server error
    Internal(String),

    /// Rate limited
    RateLimited,

    /// Procedure-specific error detail
    Detail(T),
}

// Convenience type alias for procedures with specific errors
pub type LoginError = AppError<LoginErrorDetail>;
pub type GetUserError = AppError<GetUserErrorDetail>;

#[derive(Debug, Clone, Serialize, Deserialize, Type)]
pub struct LoginErrorDetail {
    pub invalid_credentials: bool,
    pub account_locked: bool,
}

#[derive(Debug, Clone, Serialize, Deserialize, Type)]
pub struct GetUserErrorDetail {
    pub user_not_found: bool,
}
```

### Usage in Procedures

```rust
// Simple procedure — no custom error detail
#[remote(query)]
async fn get_version(ctx: &AppState) -> Result<String, AppError> {
    Ok(env!("CARGO_PKG_VERSION").to_string())
}

// Procedure with specific error detail
#[remote(command)]
async fn login(ctx: &AppState, req: LoginRequest) -> Result<AuthToken, AppError<LoginErrorDetail>> {
    match ctx.auth.login(&req.email, &req.password).await {
        Ok(token) => Ok(token),
        Err(AuthError::InvalidCredentials) => Err(AppError::Detail(LoginErrorDetail {
            invalid_credentials: true,
            account_locked: false,
        })),
        Err(AuthError::AccountLocked) => Err(AppError::Detail(LoginErrorDetail {
            invalid_credentials: false,
            account_locked: true,
        })),
    }
}

// Using convenience alias
#[remote(query)]
async fn get_user(ctx: &AppState, id: String) -> Result<User, GetUserError> {
    ctx.db.get_user(&id).await
        .ok_or(AppError::NotFound)
}
```

### Shared Error Variants as Helpers

```rust
impl<T: Serialize + Deserialize + Type> AppError<T> {
    pub fn unauthorized() -> Self { Self::Unauthorized }
    pub fn forbidden() -> Self { Self::Forbidden }
    pub fn not_found() -> Self { Self::NotFound }
    pub fn bad_request(msg: impl Into<String>) -> Self { Self::BadRequest(msg.into()) }
    pub fn internal(msg: impl Into<String>) -> Self { Self::Internal(msg.into()) }
    pub fn rate_limited() -> Self { Self::RateLimited }
    pub fn detail(d: T) -> Self { Self::Detail(d) }
}
```

### Axum Integration

`AppError<T>` implements `IntoResponse` for Axum:

```rust
impl<T: Serialize + Type> IntoResponse for AppError<T> {
    fn into_response(self) -> Response {
        let status = match &self {
            AppError::Unauthorized => StatusCode::UNAUTHORIZED,
            AppError::Forbidden => StatusCode::FORBIDDEN,
            AppError::NotFound => StatusCode::NOT_FOUND,
            AppError::BadRequest(_) => StatusCode::BAD_REQUEST,
            AppError::Internal(_) => StatusCode::INTERNAL_SERVER_ERROR,
            AppError::RateLimited => StatusCode::TOO_MANY_REQUESTS,
            AppError::Detail(_) => StatusCode::UNPROCESSABLE_ENTITY, // 422 for business logic errors
        };
        let body = serde_json::to_vec(&self).unwrap();
        (status, body).into_response()
    }
}
```

## TypeScript Side: `Result<T, E>` with Transport Errors

The generated TypeScript client returns a `Result` type that distinguishes between transport and application errors:

```typescript
// errors.ts — generated by teleport-rs

export type AppError<T = never> =
  | { type: "Unauthorized" }
  | { type: "Forbidden" }
  | { type: "NotFound" }
  | { type: "BadRequest"; message: string }
  | { type: "Internal"; message: string }
  | { type: "RateLimited" }
  | { type: "Detail"; detail: T };

export type TransportError =
  | { type: "NetworkError"; message: string }
  | { type: "SerializationError"; message: string }
  | { type: "Timeout"; message: string }
  | { type: "ServerError"; status: number; body: string };

export type RpcResult<T, E = never> =
  | { ok: true; data: T }
  | { ok: false; error: AppError<E> } // application error from Rust
  | { ok: false; transport: TransportError }; // network/serialization error
```

### Usage in SvelteKit Remote Functions

```typescript
import { query, command } from "$app/server";
import { z } from "zod";
import { users, auth } from "$lib/api/generated/client";
import type {
  AppError,
  LoginErrorDetail,
  GetUserErrorDetail,
} from "$lib/api/generated/errors";

// Query with Zod validation
export const getUserRemote = query(z.string(), async (id) => {
  const result = await users.getUser(id);

  if (result.ok) {
    return result.data;
  }

  if ("transport" in result) {
    // Transport error — network issue, retry?
    throw new Error(`Network error: ${result.transport.message}`);
  }

  // Application error — pattern match
  switch (result.error.type) {
    case "NotFound":
      throw new Error("User not found");
    case "Detail":
      // TypeScript knows detail is GetUserErrorDetail here
      throw new Error("User error: " + JSON.stringify(result.error.detail));
    default:
      throw new Error("Unexpected error");
  }
});

// Command
export const loginRemote = command(
  z.object({ email: z.string(), password: z.string() }),
  async (req) => {
    const result = await auth.login(req);

    if (result.ok) {
      return result.data;
    }

    if ("transport" in result) {
      throw new Error(`Network error: ${result.transport.message}`);
    }

    // Pattern match on application error
    if (result.error.type === "Detail") {
      const detail = result.error.detail as LoginErrorDetail;
      if (detail.invalidCredentials) {
        throw new Error("Invalid credentials");
      }
      if (detail.accountLocked) {
        throw new Error("Account locked");
      }
    }

    throw new Error("Login failed");
  },
);
```

### Helper Functions (in client library)

```typescript
// @teleport-rs/client helpers

export function isTransportError<T, E>(
  result: RpcResult<T, E>,
): result is { ok: false; transport: TransportError } {
  return !result.ok && "transport" in result;
}

export function isAppError<T, E>(
  result: RpcResult<T, E>,
): result is { ok: false; error: AppError<E> } {
  return !result.ok && "error" in result;
}

export function unwrap<T, E>(result: RpcResult<T, E>): T {
  if (result.ok) return result.data;
  if ("transport" in result) throw new Error(result.transport.message);
  throw new Error(`Application error: ${result.error.type}`);
}
```

## Error Flow Summary

```
Rust handler returns Err(AppError::Detail(LoginErrorDetail { ... }))
  ↓
Axum serializes AppError<LoginErrorDetail> as JSON with appropriate HTTP status
  ↓
TS client receives HTTP response
  ↓
If HTTP error → TransportError
If HTTP success (2xx) → parse body as T
If HTTP 4xx/5xx with AppError body → parse as AppError<E>
  ↓
Returns RpcResult<T, E>

RpcResult is discriminated union:
  { ok: true, data: T }
  { ok: false, error: AppError<E> }       // from Rust
  { ok: false, transport: TransportError } // network/protocol
```
